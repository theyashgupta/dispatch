# Support

Dispatch is a solo-maintained side project. Here's the fastest way to get unstuck.

## Setup or usage questions

Start with the [README](README.md). The [Getting started](README.md#getting-started) section covers
prerequisites, config, and first run, and Dispatch tells you at startup exactly which of `node`, `tmux`,
`ttyd`, or `claude` is missing.

Still stuck? Ask in [**Discussions**](https://github.com/theyashgupta/dispatch/discussions). That's the
right place for "how do I...", "is this supposed to...", and open-ended ideas.

## Something's broken

If Dispatch is clearly misbehaving, [open a bug report](https://github.com/theyashgupta/dispatch/issues/new/choose).
A reliable repro is the most useful thing you can bring.

## Security issues

Don't post those publicly. See [SECURITY.md](SECURITY.md) for private reporting.

## Response expectations

One maintainer, best effort. Bugs and security reports get priority. Feature discussion happens when it
happens. Thanks for your patience.
